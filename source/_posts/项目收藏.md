{% hideToggle AI 编码代理 %}
这类项目是当前最火的赛道——让 AI 直接在终端里帮你写代码、跑任务。

- [Cherry Studio](https://github.com/CherryHQ/cherry-studio) ⭐ 48.7k — AI 生产力工作室，集成智能对话、自主代理和 300+ 助手，统一接入前沿 LLM。桌面 GUI 应用，适合非终端用户高效使用 AI。亮点是开箱即用的丰富助手生态和跨平台桌面体验。

- [Goose](https://github.com/aaif-goose/goose) ⭐ 51.2k — 开源可扩展 AI 代理。超越简单的代码建议，支持安装、执行、编辑和测试，可搭配任意 LLM 使用。亮点在于高度可扩展的插件架构，面向终端和自动化场景。

- [Warp](https://github.com/warpdotdev/warp) ⭐ 63.3k — 诞生于终端的智能开发环境。将传统终端升级为 AI 驱动的 agentic 开发环境，集成了智能补全、命令搜索和工作流自动化。

- [Toad](https://github.com/batrachianai/toad) ⭐ 3.3k — 终端 AI 统一接口。用一个工具统一管理不同的 AI 服务，解决终端环境下多 AI 后端切换的痛点。

- [CC-Switch](https://github.com/farion1231/cc-switch) ⭐ 118.5k — 跨平台桌面 All-in-One 助手。统一管理 Claude Code、Codex、OpenCode、OpenClaw、Gemini CLI 等多种 AI 编码工具，一站式切换。亮点是真正做到了多工具统一管理。

- [Codex Web](https://github.com/0xcaff/codex-web) ⭐ 198 — Codex Desktop 的浏览器前端。让你通过网页界面远程控制本机的 Codex，解决了纯终端交互不够直观的问题。

- [Pi](https://github.com/earendil-works/pi) ⭐ 72.3k — AI 代理工具包，包含交互式编码代理 CLI、统一多提供商 LLM API（支持 OpenAI、Anthropic、Google 等）、Agent 运行时、TUI 终端 UI 库和 Slack 机器人。亮点是单体仓库架构，从底层 LLM 调用到上层编码代理全链路自研，供应链安全做得非常严格（依赖锁定、签名审计、shrinkwrap）。
{% endhideToggle %}

{% hideToggle AI 代理技能与框架 %}
这类项目关注 AI 代理的能力扩展和编排方式。

- [Superpowers](https://github.com/obra/superpowers) ⭐ 256.8k — 代理技能框架与软件开发方法论。提供一套完整的 agentic skills 框架，让 AI 编码工具遵循结构化的开发流程。亮点是经过实战验证的方法论，而非理论框架。

- [Matt Pocock Skills](https://github.com/mattpocock/skills) ⭐ 176.0k — 面向"真实工程"的实战代理技能集。来自 Total TypeScript 作者 Matt Pocock 的 .claude 目录，提供 /grill-me、/grill-with-docs 等需求对齐技能，解决代理与用户需求错位的核心痛点。亮点是技能小巧、可组合、易改造，适配任意模型。

- [SkillOpt](https://github.com/microsoft/SkillOpt) ⭐ 13.0k — 微软出品的文本空间技能优化器。通过轨迹驱动编辑和验证门控更新，为冻结的 LLM 代理训练可复用的自然语言技能，输出 best_skill.md 制品。解决手写 agent 技能难以调优、效果不稳定的问题。亮点是学术级方法论与可直接部署的技能制品相结合。

- [Ponytail](https://github.com/DietrichGebert/ponytail) ⭐ 85.3k — 让 AI 代理像"最懒的资深工程师"一样思考的技能。核心理念是"最好的代码是你永远不用写的代码"，引导代理优先复用、删除、减少改动。解决代理过度生成代码、引入不必要复杂度的问题。亮点是反直觉的"少即是多"工程哲学。

- [OpenCLI](https://github.com/jackwener/OpenCLI) ⭐ 26.9k — 将任意网站变成 CLI 工具。借助 AI 代理使用你已登录的浏览器，把 Web 操作转化为命令行交互。亮点是支持已登录状态的浏览器复用，绕过认证难题。

- [MemPalace](https://github.com/MemPalace/mempalace) ⭐ 57.4k — 开源 AI 记忆系统。目前基准测试表现最好的开源 AI 记忆方案，且完全免费。解决了 AI 代理缺乏长期记忆的核心问题。

- [Herdr](https://github.com/ogulcancelik/herdr) ⭐ 17.8k — 终端代理多路复用器。像一个"代理管理器"，可以同时调度多个 AI 代理协同工作，适合复杂的多步骤任务场景。

- [Awesome Persona Distill Skills](https://github.com/xixu-me/awesome-persona-distill-skills) ⭐ 4.6k — 以人物关系、纪念场景和方法论视角为主题的代理技能集合。解决的是 AI 在人际交往和情感场景下的表达能力问题。
{% endhideToggle %}

- [Ian Xiaohei Illustrations](https://github.com/helloianneo/ian-xiaohei-illustrations) ⭐ 8.1k — 中文小黑怪诞风格正文配图生成 Skill。16:9 白底手绘风格，搭配少量红橙蓝批注色彩，专为文章配图场景设计。亮点是将独特的怪诞美学风格标准化为可复用的 AI 技能，一键生成风格一致的插图。

{% hideToggle AI 提效与优化 %}
这类项目关注如何让 AI 工具跑得更快、用得更省。

- [RTK](https://github.com/rtk-ai/rtk) ⭐ 71.6k — CLI 代理，在常见开发命令的输出传入 LLM 之前进行过滤和压缩，减少 60-90% 的 token 消耗。单个 Rust 二进制文件，零依赖，支持 100+ 命令，开销 <10ms。亮点是极低性能开销和零侵入接入（只需设为代理即可生效）。

- [Headroom](https://github.com/chopratejas/headroom) ⭐ 59.7k — 在工具输出、日志和 RAG 数据传入 LLM 之前进行压缩，减少 60-95% 的 token 消耗且不影响回答质量。提供库、代理和 MCP Server 三种接入方式。亮点是显著的 token 成本节省。

- [Readme AI](https://github.com/eli64s/readme-ai) ⭐ 2.9k — AI 驱动的 README 文件生成器。自动分析项目结构和代码，生成专业规范的 README 文档，省去手动编写的繁琐工作。
{% endhideToggle %}

这类项目围绕 QQ / IM 平台的协议端、机器人框架与生态，解决消息平台接入与自动化的问题。

- 🆕 [NapCat](https://github.com/NapNeko/NapCatQQ) ⭐ 9.9k — 基于 NTQQ 的现代化 Bot 协议端实现。完整实现了大部分标准 OneBot 接口，作为初学者也能轻松上手，在低内存系统上可长时稳定运行。解决在 NTQQ 时代缺少稳定可用的 QQ 机器人协议端的问题。亮点是持续稳定开发维护、接口完整，是当前 QQ 机器人生态的事实标准之一。

- 🆕 [SnowLuma](https://github.com/SnowLuma/SnowLuma) ⭐ 605 — 将 QQ 原生会话桥接到 OneBot、WebUI 与自动化工具的远程协议框架。作为 NapCat 的现代 GUI 替代品，提供开箱即用的图形管理界面，官方在 NapCat README 中直接推荐。解决协议端缺少友好图形管理界面的问题。亮点是配套 SDK 与 MCP 包，可把 QQ 会话直接接入自动化工作流和 AI 代理。

- 🆕 [AstrBot](https://github.com/AstrBotDevs/AstrBot) ⭐ 36.5k — 易上手的多平台即时通讯 AI 代理助手与开发框架。集成 QQ、微信、Telegram、钉钉、飞书、Discord 等多种 IM 平台，统一接入各类 LLM 与插件，可作为 OpenClaw 等方案的开源替代。解决要在多个 IM 平台同时部署 AI 助手、各自适配协议繁琐的问题。亮点是多平台一套代码、插件生态丰富、自带 WebUI 可视化管理。

## Token 压缩：RTK vs Headroom

| 维度 | RTK | Headroom |
| --- | --- | --- |
| 实现语言 | Rust（单二进制，零依赖） | TypeScript |
| 压缩方式 | CLI 代理，过滤和压缩命令输出 | 库/代理/MCP Server，压缩日志和 RAG 数据 |
| 支持范围 | 100+ 常见开发命令（git、ls、cat 等） | 通用文本压缩，不限命令类型 |
| 性能开销 | <10ms | 较低 |
| 接入方式 | 设为系统代理即可生效，零配置 | 需选择库/代理/MCP 其中一种接入 |
| 压缩率 | 60-90% | 60-95% |
| 选择建议 | 追求极致性能和零配置 → **RTK** | 需要灵活接入方式或压缩 RAG 数据 → **Headroom** |
