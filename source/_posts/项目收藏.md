## AI 编码代理

这类项目是当前最火的赛道——让 AI 直接在终端里帮你写代码、跑任务。

- [Cherry Studio](https://github.com/CherryHQ/cherry-studio) ⭐ 47.0k — AI 生产力工作室，集成智能对话、自主代理和 300+ 助手，统一接入前沿 LLM。桌面 GUI 应用，适合非终端用户高效使用 AI。亮点是开箱即用的丰富助手生态和跨平台桌面体验。

- [Goose](https://github.com/aaif-goose/goose) ⭐ 46.8k — 开源可扩展 AI 代理。超越简单的代码建议，支持安装、执行、编辑和测试，可搭配任意 LLM 使用。亮点在于高度可扩展的插件架构，面向终端和自动化场景。

- [Warp](https://github.com/warpdotdev/warp) ⭐ 61.1k — 诞生于终端的智能开发环境。将传统终端升级为 AI 驱动的 agentic 开发环境，集成了智能补全、命令搜索和工作流自动化。

- [Toad](https://github.com/batrachianai/toad) ⭐ 3.2k — 终端 AI 统一接口。用一个工具统一管理不同的 AI 服务，解决终端环境下多 AI 后端切换的痛点。

- [CC-Switch](https://github.com/farion1231/cc-switch) ⭐ 93.0k — 跨平台桌面 All-in-One 助手。统一管理 Claude Code、Codex、OpenCode、OpenClaw、Gemini CLI 等多种 AI 编码工具，一站式切换。亮点是真正做到了多工具统一管理。

- [Codex Web](https://github.com/0xcaff/codex-web) ⭐ 139 — Codex Desktop 的浏览器前端。让你通过网页界面远程控制本机的 Codex，解决了纯终端交互不够直观的问题。

- [Pi](https://github.com/earendil-works/pi) ⭐ 60.2k — AI 代理工具包，包含交互式编码代理 CLI、统一多提供商 LLM API（支持 OpenAI、Anthropic、Google 等）、Agent 运行时、TUI 终端 UI 库和 Slack 机器人。亮点是单体仓库架构，从底层 LLM 调用到上层编码代理全链路自研，供应链安全做得非常严格（依赖锁定、签名审计、shrinkwrap）。

## AI 代理技能与框架

这类项目关注 AI 代理的能力扩展和编排方式。

- [Superpowers](https://github.com/obra/superpowers) ⭐ 219.2k — 代理技能框架与软件开发方法论。提供一套完整的 agentic skills 框架，让 AI 编码工具遵循结构化的开发流程。亮点是经过实战验证的方法论，而非理论框架。

- [OpenCLI](https://github.com/jackwener/OpenCLI) ⭐ 23.6k — 将任意网站变成 CLI 工具。借助 AI 代理使用你已登录的浏览器，把 Web 操作转化为命令行交互。亮点是支持已登录状态的浏览器复用，绕过认证难题。

- [MemPalace](https://github.com/MemPalace/mempalace) ⭐ 54.0k — 开源 AI 记忆系统。目前基准测试表现最好的开源 AI 记忆方案，且完全免费。解决了 AI 代理缺乏长期记忆的核心问题。

- [Herdr](https://github.com/ogulcancelik/herdr) ⭐ 4.7k — 终端代理多路复用器。像一个"代理管理器"，可以同时调度多个 AI 代理协同工作，适合复杂的多步骤任务场景。

- [Awesome Persona Distill Skills](https://github.com/xixu-me/awesome-persona-distill-skills) ⭐ 4.4k — 以人物关系、纪念场景和方法论视角为主题的代理技能集合。解决的是 AI 在人际交往和情感场景下的表达能力问题。

## AI 提效与优化

这类项目关注如何让 AI 工具跑得更快、用得更省。

- [RTK](https://github.com/rtk-ai/rtk) ⭐ 59.9k — CLI 代理，在常见开发命令的输出传入 LLM 之前进行过滤和压缩，减少 60-90% 的 token 消耗。单个 Rust 二进制文件，零依赖，支持 100+ 命令，开销 <10ms。亮点是极低性能开销和零侵入接入（只需设为代理即可生效）。

- [Headroom](https://github.com/chopratejas/headroom) ⭐ 15.0k — 在工具输出、日志和 RAG 数据传入 LLM 之前进行压缩，减少 60-95% 的 token 消耗且不影响回答质量。提供库、代理和 MCP Server 三种接入方式。亮点是显著的 token 成本节省。

- [Readme AI](https://github.com/eli64s/readme-ai) ⭐ 2.9k — AI 驱动的 README 文件生成器。自动分析项目结构和代码，生成专业规范的 README 文档，省去手动编写的繁琐工作。

## 同类项目对比

以下是功能相近的项目横向对比，方便按场景快速选择。

### AI 代理平台：Cherry Studio vs Goose

| 维度 | Cherry Studio | Goose |
|------|--------------|-------|
| 交互形式 | 桌面 GUI 应用 | 终端 CLI |
| 定位 | 通用 AI 生产力工具 | 开发者自动化代理 |
| LLM 支持 | 统一接入多家前沿模型 | 支持任意 LLM |
| 扩展方式 | 300+ 内置助手市场 | 插件架构，可自行开发 |
| 适合人群 | 非技术用户、产品经理、内容创作者 | 开发者、DevOps、自动化场景 |
| 选择建议 | 想要图形界面、开箱即用 → **Cherry Studio** | 想要终端集成、自动化流程 → **Goose** |

### Token 压缩：RTK vs Headroom

| 维度 | RTK | Headroom |
|------|-----|----------|
| 实现语言 | Rust（单二进制，零依赖） | TypeScript |
| 压缩方式 | CLI 代理，过滤和压缩命令输出 | 库/代理/MCP Server，压缩日志和 RAG 数据 |
| 支持范围 | 100+ 常见开发命令（git、ls、cat 等） | 通用文本压缩，不限命令类型 |
| 性能开销 | <10ms | 较低 |
| 接入方式 | 设为系统代理即可生效，零配置 | 需选择库/代理/MCP 其中一种接入 |
| 压缩率 | 60-90% | 60-95% |
| 选择建议 | 追求极致性能和零配置 → **RTK** | 需要灵活接入方式或压缩 RAG 数据 → **Headroom** |
